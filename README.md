**SC2026 蛋白设计模型设计简介**

**一、模型概述**

本系统采用深度学习与强化学习优化 sfGFP (绿色荧光蛋白S65T突变体)的荧光强度,通过结合ESM-2蛋白质语言模型嵌入、亮度预测模型(MLP价值网络)和双头PPO强化学习网络,在庞大的蛋白质序列空间中高效搜索高荧光强度变体。

**二、模型构建流程**

**硬件环境**

\- GPU: NVIDIA A800, 80GB显存

\- 模型: ESM-2 650M参数版本

1.  **数据预处理与特征工程**

**（1）输入数据**

\- GFP_data.xlsx : 多种GFP突变体的亮度测量数据(野生型包括avGFP, amacGFP, ppluGFP, cgreGFP)

\- 起始序列文件 : SC突变起始序列.txt (WT, PIII-4, FPbase三种sfGFP变体)

**（2）数据清洗**

①突变验证 : 验证突变字符串与野生型序列一致性

\- 支持冒号 : 和分号 ; 两种分隔符

\- 严格验证模式:验证失败则跳过样本

\- 宽松模式:验证失败仍尝试构建序列

②极低亮度过滤 : 去除亮度值 < 0.01的噪音样本

③突变计数 : 计算每条序列相对于野生型的突变数量,用于分层划分

**（3）序列编码与特征提取**

ESM-2蛋白质语言模型嵌入 :

\- 模型选择: ESM-2 650M (esm2_t33_650M_UR50D)

\- 多层特征融合: 提取第23、28、33层嵌入并拼接

\- 嵌入维度: 单层1280维 → 融合后3840维

\- Mean Pooling: 将序列级嵌入(\[seq_len, 1280\])汇聚为固定向量(\[3840\])

**（4）归一化策略**

\- Z-score归一化 : (brightness - mean) / std

\- log1p变换 : log(1 + brightness) ,处理长尾分布和零值

\- 反归一化: 预测时先反Z-score,再expm1反log变换

**（5）训练/验证集划分**

\- 划分比例: 80%训练集,20%验证集

\- 分层策略: 按突变数量分层,确保高阶突变在两个集合中都有分布

\- 随机种子: 固定42,保证实验可复现

1.  **构建荧光亮度预测模型**

作为亮度评估器,预测给定氨基酸序列的荧光强度,为强化学习系统提供即时奖励反馈。

**模型架构**

**MLP预测头设计 :**

输入层 \[3840维\] → LayerNorm → Linear\[512\] → LayerNorm → GELU → Dropout(0.3)

→ Linear\[128\] → LayerNorm → GELU → Dropout(0.3)

→ Linear\[1\] → 输出亮度预测值

**训练方式**

**监督学习训练 :**

\- 损失函数: HuberLoss (delta=1.0)

\- 残差 < 1时: 使用MSE(平滑收敛)

\- 残差 ≥ 1时: 使用MAE(对异常值鲁棒)

\- 避免长尾分布中极端值主导训练

\- 优化器: AdamW

\- 学习率: 1e-4

\- 权重衰减: 1e-5

\- 学习率调度: ReduceLROnPlateau

\- 监控指标: 验证集R²

\- 衰减因子: 0.5

\- patience: 5个epoch

\- 早停策略:

\- 监控指标: 验证集R²(推荐)或验证损失

\- patience: 10个epoch

\- 保存最佳模型checkpoint

**训练流程**

1\. 加载预处理后的ESM-2嵌入和亮度标签

2\. 按batch_size=64分批训练

3\. 每个epoch计算验证集损失和R²

4\. 学习率调度器根据验证R²调整学习率

5\. 早停机制监控验证R²,保存最佳模型

6\. checkpoint保存:包含模型权重、归一化参数、模型配置

1.  **强化学习系统核心框架**

基于当前序列状态,学习最优突变策略,生成高亮度突变序列。

**（1）双头决策网络设计**

**网络架构 :**

状态输入 \[state_dim\] → 共享编码器 → 隐藏特征 \[256维\]

↓

┌───────────────┴───────────────┐

↓ ↓

策略头 → 动作概率分布 价值头 → 状态价值估计

**共享编码器 :**

\- Linear\[state_dim → 512\] → ReLU → LayerNorm

\- Linear\[512 → 256\] → ReLU → LayerNorm

\- 输出共享特征向量\[256维\]

**策略头 :**

\- Linear\[256 → 256\] → ReLU → Linear\[256 → action_dim\]

\- Softmax → 动作概率分布(包含停止动作)

**价值头 :**

\- Linear\[256 → 256\] → ReLU → Linear\[256 → 1\]

\- 输出状态价值估计V(s)

1.  **状态空间设计**

**状态向量构成：**

**状态向量 = ESM嵌入 + 突变掩码 + 突变氨基酸特征 + 突变数**

\- ESM嵌入: \[3840维\] (原始ESM-2嵌入,不进行L2归一化)

\- 突变掩码: \[238维\] (0=未突变,1=已突变)

\- 突变氨基酸特征: \[238×20=4760维\] (one-hot编码)

\- 突变数: \[1维\] (归一化到\[0,1\])

总维度: 3840 + 238 + 4760 + 1 = 8839维

**Padding机制 :**

\- 使用 reference_length 参数统一不同长度序列的状态维度

\- 避免不同起始序列(WT/PIII-4/FPbase)长度差异导致维度不一致

**（3）动作空间设计**

**离散动作空间 :**

\- 动作类型: (位置, 氨基酸)二元组

\- 动作空间大小: seq_length × 20 + 1 (包含停止动作)

\- 标准氨基酸: 'ACDEFGHIKLMNPQRSTVWY' (20种)

**动作索引映射 :**

整数索引 → (位置, 氨基酸)二元组

action_idx → pos = action_idx // 20

→ aa = STANDARD_AA\[action_idx % 20\]

停止动作

stop_action_idx = seq_length \* 20 → (-1, "STOP")

**离散动作空间 :**

\- 动作类型: (位置, 氨基酸)二元组

\- 动作空间大小: seq_length × 20 + 1 (包含停止动作)

\- 标准氨基酸: 'ACDEFGHIKLMNPQRSTVWY' (20种)

**（4）奖励函数设计**

总奖励 = 基础亮度奖励 + 亮度提升奖励 - 突变惩罚 + 终止奖励

基础亮度奖励: (brightness + 2.0) × 3.0

\- 偏移量+2.0确保初始奖励为正(Z-score空间WT约-2.1到-1.2)

\- 权重×3放大亮度信号

亮度提升奖励: (brightness - wt_brightness) × 2.0

\- 鼓励相对于WT的任何提升(降低门槛)

突变惩罚: n_mutations × 0.5

\- 限制突变数量,鼓励少突变

终止奖励(仅在停止时):

\- 1个突变: +30.0

\- 2个突变: +20.0

\- 3个突变: +12.0

\- ≥4个突变: +5.0

\- 条件: brightness > wt_brightness

**（5）优先经验回放机制**

**普通经验池 :**

\- 容量: 10,000条经验

\- 存储格式: \[状态向量, 动作索引, 奖励, log_prob, 价值估计, 终止标志\]

**优先采样公式 :**

采样概率: P(i) = p_i^α / Σ p_j^α

其中: p_i = |TD_error(i)| + ε

参数:

\- α = 0.6 (优先级指数,0=均匀采样,1=完全优先)

\- ε = 1e-6 (避免零优先级)

**重要性采样权重 :**

w_i = (N × P(i))^(-β)

用于修正优先采样引入的分布偏差

参数:

\- β = 0.4 → 1.0 (逐渐增大,完全补偿偏差)

\- β_increment = 0.001 (每采样一步增加)

**（6）训练策略**

1\. 从起始序列(WT/PIII-4/FPbase)初始化环境

2\. 使用双头网络采样动作序列(最多8步突变或提前停止)

3\. 计算每步奖励并推入优先回放缓冲区

4\. 每10个episode从缓冲区优先采样并更新策略

5\. 保存共享策略模型(所有起始序列共用)

6\. 使用训练好的策略生成候选序列
